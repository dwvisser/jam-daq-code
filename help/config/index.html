Jam Configuration

There are two files that Jam reads when starting up. JamConfig.ini contains the properties of the current system and JamUser.ini contains the user/session preferences. If these files do not exist, Jam uses a default setup. This default setup is sufficient for offline sorting, but not for online data acquisition. Jam reads JamConfig.ini first, followed by JamUser.ini. Properties in JamConfig.ini can be overridden by JamUser.ini.

JamConfig.ini
This file tells jam what IP address, ports and to use to communicate with the front-end computer. It can also give a default sort class and default data stream definitions. JamConfig.ini also gives the default user properties, which will be overridden if the user defines them in her JamUser.ini file. Only one copy of JamConfig.ini should be used on any particular system. The file should reside in the directory defined by the property jam.home. The property jam.home is set by the -D option in the java command. For example:
java -Djam.home=/home/user/datacq/jam jam.JamMain
JamUser.ini
This file is a user defined file and tells jam what the default sort class, startup directories for opening files opening, etc., should be. The JamUser.ini file is usually placed in the user's home directory on Unix and Windows XP systems.
