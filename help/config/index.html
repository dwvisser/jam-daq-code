Jam Configuration

There are two files that Jam reads when starting upJamConfig.ini and JamUser.ini. JamConfig.ini contains the properties of the current system while JamUser.ini contains the user/session preferences. If these files do not exist, Jam uses a default setup. This default setup is sufficient for offline sorting, but not for online data acquisition. Jam reads JamConfig.ini first, followed by JamUser.ini so properties in JamConfig.ini can be overridden by defining them in JamUser.ini.

JamConfig.ini
This file tells jam what IP address, ports and to use to communicate with the front-end computer. It can also give a default sort class and default data stream definitions. JamConfig.ini also gives the default user properties, which will be overridden if the user defines them in her JamUser.ini file. Only one copy of JamConfig.ini should be used on any particular system. A number of attempts are made to read JamConfig.ini from different locations.
Order of locations from which attempts are made to read JamConfig.ini
  1. The directory defined by the property jam.home defined at startup. Defining startup property jam.home
  2. The directory Jam was started in.
JamUser.ini
This file is a user defined file and tells jam what the default sort class, startup directories for opening files opening, etc., should be. A number of attempts are made to read JamUser.ini from different locations.
Order of locations from which attempts are made to read JamUser.ini
  1. The user home directory.
  2. The directory Jam was started in.
  3. The directory defined by the property jam.home defined at startup. Defining startup property jam.home
Defining property jam.home at startup
The property jam.home is set at startup by an -D option in the java command. For example:
java -Djam.home=/home/user/datacq/jam jam.JamMain
