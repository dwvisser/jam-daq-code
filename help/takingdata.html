Jam User Manual

Online Sorting

Setting up Jam to take data takes only a few easy steps:

  1. Write a sort routine. This is done before Jam is started. In a sort routine, all the histogram and gates are defined, and you write the procedure to process each event. You will need a editor and Java compiler for this. The best approach, however, is to use an integrated development environment, such as Eclipse.
  2. Start up Jam and make sure that the VME computer (with the VxWorks operating system) is and running.
  3. Setup the sorting. Tell Jam which sorting routine to use, how to name the event and histograms files and the paths to use to write out the data and histograms files.
  4. Starting the acquisition. There are two ways to take data, just acquisition and run mode.

Setup Online Sorting

To set up online sorting you need to open the Setup|Online dialog box. There are four text fields you need to fill in. Experiment Name is used to name the data files. The event data and histogram files are named with the name of the experiment with the run number appended, e.g. AlphaTest100.hdf and AlphaTest100.evn. It is best to make the name of the experiment descriptive. Sort File (.class) is a sort routine that was written by the experimenter. There is no need to add the .class extension. Path for Histograms and Path for Events are the absolute paths for storage of histograms and events.

Starting and stopping the acquistion system

To start and stop the data acquisition without saving the events, use the menu items Control|start and Control|stop.

Running: taking event data

To save the event data you need to begin a run. Open the Control|Run... dialog box to do this. Begin will open a new event file and start acquisition. The event file will be create in the directory specified in the Setup|Online dialog box. End will stop acquisition, close the event file, and write out a HDF file which if the appropriate boxes are checked, will contain the histogram, gates, and scalers.
