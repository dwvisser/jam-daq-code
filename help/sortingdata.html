Jam User Manual

Offline Sorting

Setting up Jam to sort event data takes only a few easy steps:

  1. Write a sort routine. This is done before Jam is started. In a sort routine, all the histogram and gates are defined, and you write the procedure to process each event. You will need a editor and Java compiler for this. The best approach, however, is to use an integrated development environment Eclipse. The IDE we develop Jam on is Eclipse. 
  2. Start up Jam.
  3. Setup of the offline sorting. This entails loading the sort routine and event streams you would like to use.
  4. Read in previously defined gates if needed.
  5. Specify which files to sort and begin the sort.
  6. Save the sorted spectra.

Setup of the sorting

Open up Setup|Offline. Load your sorting class to use, and event streams for input and output.

Read in previously defined gates

 If you needed read in a hdf file that has previously defined gates in it, using File|Reload.
You might also need to zero all histograms, use Histograms|Zero

Specify which files to sort and begin sort

 Open the sort dialog Control|Sort where you specify which files, (runs for tape) to sort, and begin the sort. Select the list of files you want to sort using: Add file, Remove file, Remove All.
If this list is more than a few files long you might want to save the list. You can save this list of files using Save list and subsequently load it using Load list
To start the sort click on Begin. To stop the sor click on End/Cancel, not this is not a pause, it kill the sort process. After ending a sort you will have to setup to sort again.

Saving sorted spectra

 Save the sorted spectra and gates in a hdf file with File|Save.
