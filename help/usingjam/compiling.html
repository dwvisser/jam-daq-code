Compiling Sort Routines and Scripts

This example given is for Unix-based systems like Linux and MacOS X. If you're using Windows, separate folders in the classpath with a semicolon instead of a colon. Let's assume you have a sort routine, sort.MySort, in the file MySort.java, which is in the directory /home/daq/user/classes/sort, and your copy of Jam.jar resides in JAMLIB. To compile it, you issue the following commands from the shell:

  1. cd /home/daq/user/classes/sort
  2. javac -classpath JAMLIB:/home/daq/user/classes Mysort.java

This will create a file MySort.class in /home/daq/sortfiles. We have assumed the path to javac is in your PATH environment variable.

Typical JAMLIB path for different Jam Distributions

EnvironmentLocation of library
Windows"C:\\Program\ Files\\Jam\ version#\\lib\\Jam.jar"
LinuxINSTALL_DIR/jam-version#/Jam.jar
MacOS XINSTALL_DIR/jam-version#/Jam.app/Contents/Resources/Java/Jam.jar

In the Windows case, you need to enclose the path in quotation marks and preface every space and backslash you normally type with a backslash.
