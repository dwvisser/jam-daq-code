Jam Manual

Jam Configuration

There are two files that jam reads when starting up. One file to obtain the properties of the current system and the other get the user preferences. These files are JamConfig.ini and JamUser.ini. If these files do not exist jam resorts to its build in default setup. This default setup is sufficient for offline sorting but not for online data taking. Jam first read in JamConfig.ini and then JamUser.ini so properties in JamConfig.ini can be overridden by JamUser.ini.
  • JamConfig.ini This file tells jam what IP address, ports and to use to communicate with the front-end computer. And can give the default sort class and input streams. Only one copy of this file should be needed per system. The JamConfig.ini file should reside in the jam home directory defined by the property jam.home. JamConfig.ini also give the default user properties, those that will be overridden if the user defines them in her JamUser.ini file. (The property jam.home is set by the -D option in the java command. See jam.ksh for more information.) For example:
    java -Djam.home=/home/user/jam jam.Jamdisp
  • JamUser.ini This file is a user defined file and tells jam what the default sort class, startup directories for opening files opening, etc., should be. The JamUser.ini file should be in the user home directory.

Example JamConfig.ini file

Included in your download.
!Configuration file for Jam
host.IP=hobbes
host.portSend=5003
host.portRecv=5002
target.IP=calvin
target.port=5002
host-data.IP=calvin-data
host-data.portRecv=10205
sort.class=sort.Test
event.inpath=/home/jam
event.outpath=/home/jam
event.instream=jam.sort.L002InputStream
event.outstream=jam.sort.L002OutputStream
tape.dev=/dev/rmt/0
browser.path=/usr/local/netscape
docs.path=docs
camac.path=/JAM/CAMAC/CNAFS

Download it if you like.

Parameter definitions

For the follow explanation the host is the computer Jam runs on, while the target is the computer in the VME crate.

host.IP
IP address of the socket to used to communticate with the VME crate
host.portSend
socket send port number used to communicate with the VME crate
host.portRecv
socket receive port number used to communicate with the VME crate
target.IP
IP address of the socket on the VME computer that is used to communicate with the host
target.port
socket receive port number on the VME computer that is used to communicate with the host
host-data.IP
IP address of the socket to used to receive data from VME crate
host-data.portRecv
socket receive port number used to receive data from the VME crate
sort.class
default class jam will use for sorting
event.instream
default event data input stream jam will use for sorting data (offline)
event.outstream
default event data output stream jam will use for sorting data (offline)
event.inpath
default event input data files path
event.outpath
default event ouput data file path
hist.path
default path jam will use for saving and opening histogram files
browser.path
path to browser java will use to open help files
docs.path
path to jam help documentation
tape.dev
the default tape device
camac.path (no longer used)
the path jam will write the camac cnafs to and from which the VME target computor will read them.

Example JamUser.ini file

NOTE: Properties listed in JamUser.ini will always override those listed in JamConfig.ini. Include with your download.
!Initialization file for Jam
camac.path=/home/jam/camac
exp.name=test
sort.path=/home/jam
event.path=/home/jam
hist.path=/home/jam

Download it if you like.

Parameter definitions

The host is the computer Jam runs on, while the target is the computer in the VME crate.

camac.path
default path jam will use for saving and opening camac CNAF files
exp.name
default name experimental name jam will use (online)
sort.file
default sort file jam will load
hist.path
default path jam will use for saving and opening histogram files
event.path
default path jam will use for saving and opening event files
